# Purpose

The Evidence & Reporting Engine is responsible for transforming validated compliance information into professional, explainable and evidence-backed outputs.

Its purpose is not merely to generate reports.

Its purpose is to communicate operational truth.

Every report should be traceable to supporting evidence.

Every conclusion should be explainable.

---

# Philosophy

Reports should never become the source of truth.

Evidence remains the source of truth.

Reports are simply views of evidence at a point in time.

This distinction allows reports to evolve while preserving trust in the underlying data.

---

# Implemented Capabilities

The Reporting Engine implements:

* **REP-001**: Driver Daily Reports
* **REP-002**: Fleet Reports
* **REP-003**: Compliance Reports
* **REP-004**: PDF Export
* **REP-005**: CSV Export
* **REP-006**: Audit Packs
* **SYS-003**: Background Jobs (Report Generation)

It is **not** responsible for:

* evidence collection
* compliance calculations
* timeline generation
* AI reasoning

# Report Lifecycle

Every report follows the same lifecycle.

```text
Request

↓

Collect Evidence

↓

Validate Sources

↓

Generate Report

↓

Generate Evidence References

↓

Publish

↓

Archive

↓

Retrieve
```

Reports should always be reproducible.

---

# Report Identity

Every generated report should receive:

* Report ID
* Version
* Generation Date
* Generated By
* Company
* Ruleset Version
* Processing Version

Reports become historical records.

---

# Report Versioning

Reports generated today may differ from reports generated in the future due to:

* legislation changes
* ruleset updates
* improved processing
* corrected evidence

Older reports should remain reproducible using the versions that existed when they were created.

# Explainability

Every significant conclusion should answer:

Why is this shown?

Which evidence supports it?

Which rule applies?

How was it calculated?

Where can the original evidence be viewed?

---

# Audit Support

Reports should support investigations by preserving:

processing version

# Performance Goals

support large fleets

Generation speed should never compromise correctness.

# Engineering Rules

The Reporting Engine must never:

calculate compliance

modify evidence

invent explanations

discard evidence references

hide uncertainty

Reports communicate information.

They do not create it.

---

# Definition of Done

The Reporting Engine is complete when:

reports generate successfully

Evidence Packs work

versioning exists

evidence references exist

audit information is preserved

security controls pass

branding works

scheduled reports work

exports remain reproducible
