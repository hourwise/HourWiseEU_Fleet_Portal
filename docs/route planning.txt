The reason is simple: Atlas is only useful if the underlying system already knows:

Driver X is on Shift Y
Vehicle Z is assigned
Job 2 is delayed
Next leg is 1h 15m
Driver has 45m driving left
Break is required

Then Atlas can compile that into:

“This route is now at risk. The driver should take a compliant break before leaving Job 2, and the manager should warn Job 3 of a late ETA.”

So I’d build it like this:

Phase 1 — Rules engine, no AI
The system creates warnings using deterministic logic.

Phase 2 — Atlas summaries
Atlas explains what the rules engine already found.

Phase 3 — Atlas recommendations
Atlas suggests actions.

Phase 4 — Atlas assistant
Managers can ask questions.

This keeps HourWise trustworthy. The legal/compliance logic should come from your own calculations, not from AI guessing.
